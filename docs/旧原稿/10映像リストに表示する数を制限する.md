映像リストに表示する数を制限する

該当commit

次に映像リストに表示する数を減らしていきます。

やる事

表示データを20→10にする

データの制限について

本来なら取得の段階でレコード数を制限するのが望ましいです。表示する可能性のあるすべてのデータを1回で取得すると初回のローディングが重くなりWebサービスとしては推奨出来ません。

基本的には以下の方針を取ります。

表示に必要なレコード数だけ取得する

データ量が増える場合はページネーションなどを用いて、回数を分けて取得する

今回はNetlifxクローンというSPA(シングルページアプリケーション)の構成になっていて、ページ単位での画面切り替わりはない為、ページネーションは使用しません。

また、TMDBがレコード数を指定して取得する事が出来ず、デフォルトで20個が返される仕様となっております。

そこで、レスポンスを取得してからRow/useProps.tsにてデータ数を減らしていきます。

コード解説

Row/useProps.ts

import { useState } from "react";
import axios from "../../axios";
import { useQuery } from "react-query";
import { Movie } from "../../type.ts";
import { requests } from "../../request.ts";

export const useProps = (fetchUrl: string, title: string) => {
  const [trailerUrl, setTrailerUrl] = useState<string | null>("");
  const fetchData = async () => {
    const request = await axios.get(fetchUrl);
    
    // データ数を10個に制限
    return request.data.results.slice(0, 10).map((movie: Movie) => ({
      id: movie.id,
      name: movie.name,
      poster_path: movie.poster_path,
      backdrop_path: movie.backdrop_path,
    }));
  };

  const { data: movies, isLoading } = useQuery(`${title}/movies`, fetchData);

  const handleClick = async (movie: Movie) => {
    if (trailerUrl) {
      setTrailerUrl("");
    } else {
      const moviePlayUrl = await axios.get(requests.fetchMovieVideos(movie.id));
      setTrailerUrl(moviePlayUrl.data.results[0]?.key);
    }
  };

  return {
    movies,
    trailerUrl,
    handleClick,
    isLoading,
  };
};

変更は1箇所です！

return request.data.results.slice(0, 10).map((movie: Movie) => ({
      id: movie.id,
      name: movie.name,
      poster_path: movie.poster_path,
      backdrop_path: movie.backdrop_path,
    }));

配列に対してsliceを使い、1~10個目までのみを抽出しています。

これにてLayout.tsx側にわたるデータ数は20→10になります。
